# QUALITY_SCORE.md

## 1. 등급 체계

| 등급 | 기준 | QuSign 적용 영역 |
|---|---|---|
| S | 장애 시 서비스 즉시 불신 | PQC 암호화 레이어 |
| A | 핵심 기능 저하 | 인증, 서명 플로우 API |
| B | 일부 사용자 영향 | 문서 관리, 이메일 연동 |
| C | 불편 정도 | 프론트엔드 UI |
| D | 실험적, 폐기 가능 | PoC, 벤치마크 스크립트 |

## 2. 등급별 요구사항

| 항목 | S | A | B | C | D |
|---|---|---|---|---|---|
| 단위 테스트 | 90%+ | 70%+ | 40%+ | - | - |
| 통합 테스트 | 필수 | 필수 | 권장 | - | - |
| 타입 체크 | strict | strict | normal | normal | - |
| 모니터링 | 필수 (6단계) | 필수 (6단계) | 권장 | - | - |
| 롤백 절차 | 필수 | 필수 | 권장 | - | - |

## 3. 본 프로젝트의 영역별 등급

| 영역 | 등급 | 사유 |
|---|---|---|
| `PqcSignatureService` (ML-DSA 키생성/서명/검증) | **S** | 암호화 오류 = 서비스 신뢰도 즉시 붕괴 |
| `PdfSignatureService` (서명 삽입/추출) | **S** | 서명값 손상 시 법적 효력 없음 |
| 인증 (회원가입, JWT, 개인키 암호화) | **A** | 키 노출 시 모든 사용자 영향 |
| 서명 플로우 API (요청/실행/검증) | **A** | 핵심 비즈니스 로직 |
| 문서 관리 (업로드/다운로드/목록) | **B** | 재시도 가능, 데이터 손실 없음 |
| 이메일 연동 (SES) | **B** | 실패 시 수동 링크 전달로 대체 가능 |
| 프론트엔드 (Vue 3) | **C** | 백엔드 정합성이 더 중요 |
| 벤치마크 스크립트, PoC | **D** | main 직접 머지 금지 |

## 4. S/A 영역 특별 규칙

- 자율 에이전트의 자동 변경 금지 → 항상 플랜 → 사용자 승인 → 구현
- 변경 시 반드시 단위 테스트 + 통합 테스트 통과 확인
- 리뷰 없이 main 머지 금지

## 5. 등급 변경 절차

1. `docs/design-docs/`에 ADR 작성
2. 등급 변경 이유와 영향 범위 명시
3. 이 파일 갱신 + `harness/CHANGELOG.md` 기록
