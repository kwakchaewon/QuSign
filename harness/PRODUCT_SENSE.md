# PRODUCT_SENSE.md

## 1. 제품 비전

> 양자 컴퓨터 시대에도 안전한 전자서명 — NIST PQC 표준(ML-DSA)을 일반 사용자도 쉽게 쓸 수 있는 SaaS 서비스.

## 2. 핵심 사용자

| 페르소나 | 가장 중요한 가치 |
|---|---|
| 보안 의식 있는 개인/프리랜서 | "RSA가 깨져도 내 서명은 유효하다"는 안심 |
| 클라우드 보안 엔지니어 채용 담당자 | PQC 실전 경험을 갖춘 후보자 포트폴리오 |

## 3. 의사결정 우선순위 (상충 시 위→아래)

1. **암호화 정확성** — ML-DSA 검증이 틀리면 서비스 존재 의미 없음
2. **보안** — 개인키 노출은 서비스 즉시 종료 사유
3. **단순함** — 사용자가 PQC를 몰라도 쓸 수 있어야 함
4. **성능** — ML-DSA 키 크기가 크므로 UX 허용 범위 내에서 타협 가능
5. **수익화** — 5단계 이후 고려

## 4. 안 하기로 한 것 (명시적 비-목표)

- 기존 RSA/ECDSA 호환 — PQC 전용으로만 간다
- 모바일 앱 — 웹 MVP 검증 후 검토
- 엔터프라이즈 계약 — 개인 플랜 안정화 후
- 다중 서명 알고리즘 선택 — ML-DSA 단일 고정 (5단계 이후 SLH-DSA 검토)

## 5. 팀 / 책임자

- 솔로 프로젝트 (곽채원)
- 재직 중 사이드 프로젝트 — 주당 가용 시간 제한 있음

## 6. 비즈니스 제약

| 항목 | 제약 |
|---|---|
| 인프라 비용 | AWS 월 $30 이하 (초기) |
| 개발 기간 | 12개월 |
| 규정 준수 | 전자서명법 검토 필요 (5단계 수익화 전) |
| 성능 목표 | 서명 API P95 < 3초 (ML-DSA 키 크기 감안) |
